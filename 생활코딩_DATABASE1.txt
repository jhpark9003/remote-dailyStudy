# 생활코딩_DATABASE1

데이터를 저장하고 꺼낼 수 있는, 우리가 선택한 첫번째 수단은 file.
file의 장점
 - 배우기 쉽다
 - 운영체제마다 파일기능을 제공하기 때문에 어디에서나 사용가능
 - 이메일이나 메신저를 이용해서 간편하게 전송 가능
file의 단점
 - 성능, 보안, 편의성에 한계를 가지고 있다.


file이 가진 한계를 극복하기 위해서 고안된 전문화된 소프트웨어가 database.
데이터베이스를 이용하면 소중한 데이터를 안전하고 편리하고 빠르게 보관하고 사용할 수 있다.
세상에는 다양한 데이터베이스 제품이 존재한다.
MySQL, Oracle, SQL Server, PostgreSQL, MongoDB 등

다양한 데이터베이스를 관통하는 가장 본질적인 기능들을 살펴보자.

데이터베이스는 매우 방대한 기능을 가지고있는 정보 도구이다.
데이터베이스 기능이 방대한 것은 데이터 관련해서 일어날 수 있는 일들이 참으로 많기 때문이다.

   -input-       -output-
    Create        Read
    Update
    Delete

File -> Spreadsheet -> Database

# 제품별 특징
Oracle
- 오랫동안 데이터베이스 시장에서 군림해왔던 절대강자
- 주로 관공서, 대기업에서 사용
- 매우 비쌈
- 데이터베이스 기술지원에 대한 컨설팅 비용이 비싸다.

MySQL
- 무료, 오픈소스
- 관계형 데이터베이스를 이용하고 싶지만, 개인적이거나 자금이 부족한 경우 적합

MongoDB
- 관계형 데이터베이스가 아니다.
- 2010년부터 RDBMS가 아닌 다양한 DB가 쏟아져 나왔다. NoSQL이라는 흐름이 나타났다.
